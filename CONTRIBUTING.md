# Contributing to the project

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an Issue on GitHub. Include:
- A clear and descriptive title.
- Steps to reproduce the bug.
- What you expected to happen vs. what actually happened.
- The error message from the terminal (if any).

### Suggesting Enhancements
Have an idea for a new feature? Open an Issue and tag it as an enhancement. Describe how the feature should work and why it would be useful for the users.

### Code Contributions (Pull Requests)
If you want to write code and add a feature or fix a bug yourself, follow these steps:

1. **Fork the repository** (click the "Fork" button at the top right of the repo page).
2. **Clone your fork** locally:
   `git clone https://github.com/YourUsername/sa-mp-automatic-war-evidence.git`
3. **Create a new branch** for your feature or bugfix:
   `git checkout -b feature/your-awesome-feature`
4. **Make your changes** in the code.
5. **Test your changes** to ensure everything still works.
6. **Commit your changes** with a clear message:
   `git commit -m "Add feature X" `
7. **Push to your branch**:
   `git push origin feature/your-awesome-feature`
8. **Open a Pull Request** on the original repository.

## Pull Request Guidelines
- Ensure your code is clean and readable.
- Add comments where necessary.
- Update the `README.md` if your changes affect how the script is used.
